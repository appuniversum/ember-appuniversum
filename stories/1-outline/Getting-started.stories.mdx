# Getting started

## Installation

You can install the [ember-appuniversum](https://www.npmjs.com/package/@appuniversum/ember-appuniversum) addon with ember-cli:

`ember install @appuniversum/ember-appuniversum`

## Set up the build pipeline

### SCSS

`ember-appuniversum` uses SCSS so you will need to add a SCSS compiler to your project. This can be done by installing [ember-cli-sass](https://www.npmjs.com/package/ember-cli-sass).

### Autoprefixer

`ember-appuniversum` doesn't use any CSS vendor prefixes in its code and it depends on `autoprefixer` to add those when needed. Make sure to set up autoprefixer in your project.

The easiest way to do this in an ember project is by installing the [`ember-cli-autoprefixer` addon](https://github.com/kimroen/ember-cli-autoprefixer).

## Customising the SCSS for your project

Then load the SCSS in your `app.scss` file:

```css
@import "@appuniversum/ember-appuniversum/styles";
```

> You may need to configure your SCSS compiler so that it looks at the `node_modules` folder when importing files.
> For ember-cli-sass this can be done by using the `includePaths` option.
> ```js
> // ember-cli-build.js
> var app = new EmberApp({
>  sassOptions: {
>    includePaths: ["node_modules"],
>  },
>});
>```

If you want more control over the order that SCSS partials are rendered you can import them as groups.

```css
@import "@appuniversum/ember-appuniversum/styles/a-settings";
@import "@appuniversum/ember-appuniversum/styles/a-tools";
@import "@appuniversum/ember-appuniversum/styles/a-generic";
@import "@appuniversum/ember-appuniversum/styles/a-elements";
@import "@appuniversum/ember-appuniversum/styles/a-objects";
@import "@appuniversum/ember-appuniversum/styles/a-components";
@import "@appuniversum/ember-appuniversum/styles/a-plugins";
@import "@appuniversum/ember-appuniversum/styles/a-utilities";
```
